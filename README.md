# cluster_k3s

Установщик кластера k3s с поддержкой множественных окружений и шаблонов развертывания.

## 📋 Описание проекта

Проект предназначен для автоматизированного развертывания кластера k3s в различных окружениях с использованием инфраструктуры как кода (IaC) и конфигурационного менеджмента.

## 🚀 Быстрый старт

Подробное руководство по настройке и запуску смотрите в документации:

- **[Полное руководство по настройке](docs/setup_guide.md)** - детальная инструкция по всем этапам
- **[Шпаргалка по запуску](docs/quickstart.md)** - краткое руководство для быстрого старта

### Основные шаги:

1. **Настройте переменные** в `terraform/proxmox.tfvars` или `terraform/yandex.tfvars`
2. **Создайте шаблон ВМ** в Proxmox (подробно в [setup_guide.md](docs/setup_guide.md))
3. **Запустите установку**:
   ```bash
   cd terraform
   terraform init
   terraform apply -var-file="proxmox.tfvars" -var-file="templates/simple.tfvars"
   ```

## Шаблоны развертывания

### 1. Simple (Простой)
- **Мастер-ноды:** 1
- **Worker-ноды:** N (добавляются по мере необходимости)
- **Use case:** Разработка, тестирование, небольшие проекты

### 2. Middle (Средний)
- **Мастер-ноды:** 3 (HA конфигурация)
- **Worker-ноды:** N (добавляются по мере необходимости)
- **Use case:** Production окружения с требованиями к отказоустойчивости

### 3. Custom (Пользовательский)
- **Мастер-ноды:** Настраиваемое количество (1, 3, 5...)
- **Worker-ноды:** Настраиваемое количество
- **Use case:** Специфические требования к масштабу и архитектуре

## Поддерживаемые окружения

### 1. Proxmox VE Cluster
- Виртуализация на базе Proxmox
- Локальное развертывание
- Полный контроль над инфраструктурой

### 2. Yandex Cloud
- Облачная инфраструктура
- Масштабируемость
- Pay-as-you-go модель

## Переменные окружения и конфигурация

### Глобальные переменные
- `ENVIRONMENT` - целевое окружение (proxmox|yandex)
- `TEMPLATE` - шаблон развертывания (simple|middle|custom)
- `CLUSTER_NAME` - имя кластера
- `K3S_VERSION` - версия k3s

### Переменные шаблонов
- `MASTERS_COUNT` - количество мастер-нод
- `WORKERS_COUNT` - количество worker-нод
- `VM_SPECS` - спецификации VM (CPU, RAM, Disk)

## 📋 Требования к инфраструктуре

### Proxmox VE
- Proxmox VE кластер версии 7.x или выше
- Доступ к API Proxmox
- **Предварительно настроенные шаблоны VM (cloud-init)** - см. [setup_guide.md](docs/setup_guide.md)
- Сетевая конфигурация (мост vmbr0 или другой)

### Yandex Cloud
- Аккаунт в Yandex Cloud
- Сервисный аккаунт с необходимыми правами (editor, vpc.admin, compute.admin)
- Предварительно настроенный folder и VPC
- Лимиты на создание ресурсов

## 🎯 Следующие шаги

После настройки проекта вы можете:

1. **Настроить переменные** в файлах `.tfvars` под вашу инфраструктуру
2. **Создать шаблон ВМ** в Proxmox по инструкции из [setup_guide.md](docs/setup_guide.md)
3. **Развернуть тестовый кластер** с помощью simple шаблона
4. **Проверить работу кластера** и при необходимости настроить под свои нужды
5. **Развернуть production кластер** используя middle или custom шаблон
