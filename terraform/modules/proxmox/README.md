# Proxmox Module for K3S Cluster

## Description
This module creates Virtual Machines in Proxmox VE for a K3S cluster using the `telmate/proxmox` provider.
It supports cloud-init for automatic configuration of hostname, SSH keys, and network settings.

## Requirements
- Proxmox VE 7.x or 8.x
- Terraform >= 1.0
- Proxmox API Token with sufficient permissions
- A pre-configured VM Template with cloud-init support

## Variables

| Name | Description | Type | Default | Required |
|------|-------------|------|---------|:--------:|
| `proxmox_node` | Name of the Proxmox node | `string` | `"pve"` | no |
| `template_vm_id` | ID of the source VM template | `number` | n/a | yes |
| `vm_prefix` | Prefix for VM names | `string` | `"k3s"` | no |
| `master_count` | Number of master nodes to create | `number` | `1` | no |
| `worker_count` | Number of worker nodes to create | `number` | `0` | no |
| `cpu_cores` | Number of CPU cores per VM | `number` | `2` | no |
| `memory_mb` | Amount of memory in MB per VM | `number` | `4096` | no |
| `disk_size_gb` | Size of the root disk in GB | `number` | `30` | no |
| `network_bridge` | Network bridge name | `string` | `"vmbr0"` | no |
| `ip_range_start` | Start IP address for static assignment | `string` | n/a | yes |
| `gateway` | Default gateway IP | `string` | n/a | yes |
| `cidr` | CIDR notation for subnet (e.g., /24) | `string` | `"/24"` | no |
| `dns_servers` | List of DNS servers | `list(string)` | `["8.8.8.8", "1.1.1.1"]` | no |
| `ssh_public_key` | SSH public key for cloud-init | `string` | n/a | yes |
| `user` | Username for cloud-init user | `string` | `"ubuntu"` | no |
| `tags` | Tags to apply to VMs | `list(string)` | `["k3s", "terraform"]` | no |

## Outputs

| Name | Description |
|------|-------------|
| `master_nodes` | List of maps containing master node details (name, ip, id) |
| `worker_nodes` | List of maps containing worker node details (name, ip, id) |
| `all_nodes` | Combined list of all nodes |
| `inventory_hosts` | Ansible inventory compatible string |

## Usage

```hcl
module "proxmox_k3s" {
  source = "./modules/proxmox"

  proxmox_node     = "pve"
  template_vm_id   = 9000
  vm_prefix        = "k3s-dev"
  
  master_count     = 1
  worker_count     = 2
  
  cpu_cores        = 2
  memory_mb        = 4096
  disk_size_gb     = 30
  
  network_bridge   = "vmbr0"
  ip_range_start   = "192.168.1.100"
  gateway          = "192.168.1.1"
  cidr             = "/24"
  
  ssh_public_key   = file("~/.ssh/id_rsa.pub")
  user             = "ubuntu"
}

output "cluster_nodes" {
  value = module.proxmox_k3s.all_nodes
}
```

## Cloud-Init Configuration
The module automatically configures each VM with:
- Hostname matching the node role and index
- Static IP address based on the provided range
- SSH authorized keys
- User creation
- Network configuration

## Notes
- Ensure the source template has cloud-init installed and configured
- The module reserves IPs sequentially starting from `ip_range_start`
- VM IDs are automatically generated by Proxmox unless specified otherwise
